# Documentation QA Checklist

## Scope

Docs/release being reviewed.

## Accuracy

- [ ] Instructions match current product behavior
- [ ] Setup/configuration steps were tested
- [ ] Troubleshooting steps were checked
- [ ] Screenshots/examples are current where used
- [ ] Links work

## Completeness

- [ ] User docs exist where required
- [ ] Admin docs exist where required
- [ ] Operator docs/runbooks exist where required
- [ ] Release notes exist where required
- [ ] Known limitations are documented

## Safety / Trust

- [ ] Security claims are accurate
- [ ] Privacy/data handling claims are accurate
- [ ] Compliance claims are approved where present
- [ ] Destructive/risky actions include warnings and recovery guidance

## Accessibility / Usability

- [ ] Language is clear
- [ ] Terminology is consistent
- [ ] Headings and structure are usable
- [ ] Images have useful alt text where applicable
